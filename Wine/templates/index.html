Here you will be able to predict the quality of your wine if you provide the following input.

Your input to provide to the /predict endpoint with the POST method should be formatted with the following information

Input will be JSON data with the template:
{wine : [x1,x2,x3,x4,x5,x6,x7,x8,x9,x10,x11]} with the following parameters in order:
x1 : fixed acidity
x2 : volatile acidity
x3 : citric acid
x4 : residual sugar
x5 : chlorides
x6 : free sulfur dioxide
x7 : total sulfur dioxide
x8 : density
x9 : pH
x10 : sulfates
x11 : alcohol
Multiple arrays can be provided, separated by a comma (e.g: [x1,x2,x3,x4,x5,x6,x7,x8,x9,x10,x11],[x1,x2,x3,x4,x5,x6,x7,x8,x9,x10,x11])

The expected output will be with the following format:
{'prediction': [y1]}
y1 being the prediction result. There will be as many predictions as arrays provided as input

To test the end point you can either
- Use the Test_Endpoint notebook provided in the github repository, the default url is set to run the app locally, replace with "http://wine-app-mcheraki.herokuapp.com/predict" to test the API.
- Use curl with the following command :"curl -i -H "Content-Type: application/json" -X POST -d '{"input": [[x1,x2,x3,x4,x5,x6,x7,x8,x9,x10,x11]]}' http://wine-app-mcheraki.herokuapp.com/predict"
